# Игровые правила и автомат состояний

Вся логика в `core/GameController.gd` (авторитетная, живёт только на
сервере/хосте) + `core/GameRules.gd` (чистые статические расчёты).

## Фазы партии (`phase`)

| Константа | Значение |
|---|---|
| `PHASE_LOBBY` | Лобби, игроки жмут "готов", хост может стартовать |
| `PHASE_PLAYING` | Партия идёт |
| `PHASE_ENDED` | Остался один игрок не в банкротстве — он победитель |

## Фазы хода (`turn_phase`)

```
                 ┌───────────────────┐
                 │ awaiting_jail      │◄──── игрок начал ход в тюрьме
                 └─────────┬──────────┘
                            │ pay / card / wait(3 раза)
                            ▼
   ┌────────────────────────────────────────┐
   │            awaiting_roll                │◄─┐
   └─────────────────┬────────────────────────┘  │
                       │ resolve_roll()            │ ход завершён
        ┌──────────────┼───────────────┐           │ (_finish_turn)
        ▼              ▼               ▼           │
 клетка свободна   клетка занята   особая клетка    │
        │           (аренда)       (налог/карта/…)  │
        ▼                                            │
 awaiting_purchase                                    │
   │  buy / decline                                   │
   ├─ auction_enabled=false → сразу ход дальше ────────┤
   └─ auction_enabled=true  → awaiting_auction          │
         │ все сделали ставку/пас                       │
         └────────────────────────────────────────────► │
                                                          │
      если не хватило денег на аренду/налог/долг ─────►  │
                                                          │
                managing_assets (продать/заложить/       │
                банкротство, пока долг не погашен) ──────┘
```

`TURN_ENDED` — терминальное состояние хода, когда `PHASE_ENDED`.

## Ход подробно

1. **`resolve_roll(peer_id)`** — один кубик (1–6), перемещение по кольцу
   (`posmod`, доска зациклена). Если перешагнули СТАРТ — зачисляется
   `pass_start_reward` (по умолчанию 200 000).
2. **`_resolve_landing`** — по типу клетки:
   - `property`/`railroad`/`utility` без владельца → `awaiting_purchase`;
   - занята чужим и не заложена → списывается рента (`GameRules.calculate_rent`),
     ход завершается сразу;
   - `tax` → фиксированная сумма в банк;
   - `chance`/`treasury` → тянется карта из соответствующей колоды
     (`CardDeck`, тасуется через `RandomNumberGenerator` с сидом при `setup`);
   - `corner_go_to_jail` → игрок отправляется в тюрьму;
   - любая другая клетка (в т.ч. добавленная модом) → эмитится хук
     `EventBus.run_hook("on_land", payload)`, мод может вернуть `effects`
     (список `credit`/`debit`/`move_to`/`send_to_jail`), которые применяет
     `_apply_effects`. Так DLC добавляет новые типы клеток без правки ядра.

## Рента (`GameRules.calculate_rent`)

- **Железная дорога (`railroad`)**: рента зависит от количества дорог у того
  же владельца, `rent_table` индексируется по `count - 1`.
- **Коммуналка (`utility`)**: `dice_value * multiplier_one` (одна) или
  `dice_value * multiplier_two` (обе).
- **Обычная собственность (`property`)**: базовая рента из `rent_table[improvements]`.
  Если построек ещё нет (`improvements == 0`), применяется **плавный рост
  ренты** от владения несколькими полями группы до полной застройки:
  `rent *= rent_growth ^ (owned_in_group - 1)` (см. `rent_growth_multiplier`).
  Это вместо классического разового "×2 при полной группе" — рента растёт
  постепенно с каждым купленным полем группы. `rent_growth = 1.0` (по
  умолчанию) отключает эффект полностью.
- Заложенная (`mortgaged`) собственность ренты не берёт.

## Покупка / Аукцион

- `request_buy` — списывает `price`, если хватает денег; иначе ход не
  завершается автоматически (**баг-репеллент**: `ledger.debit` вернул бы
  `false`, но UI не должен предлагать "Купить", если денег не хватает —
  сервер всё равно перепроверяет).
- `request_decline_purchase`:
  - если `auction_enabled = false` в ruleset'е (в `classic.json` — так и есть)
    — поле просто остаётся у банка, ход передаётся дальше;
  - если `true` — начинается **закрытый одноразовый аукцион** (`AuctionState`):
    каждый активный игрок либо ставит (`amount > 0`), либо пасует
    (`amount = 0`); ставки друг от друга скрыты до `resolve()`; выигрывает
    наибольшая ставка, `0` — если все пасанули (поле остаётся у банка).

## Тюрьма

- `jail_max_turns` (по умолчанию 3) попыток `wait`; если после 3-й всё ещё
  не выпал шанс выйти бесплатно (в этой реализации `wait` просто считает
  попытки и после лимита списывает штраф принудительно) — штраф `jail_fine`
  списывается автоматически, а при нехватке денег уходит в `managing_assets`
  с `resume: "jail_release"`.
- `pay` — сразу штраф и выход.
- `card` — тратит "карту освобождения" (`get_out_cards`), если есть.

## Управление активами / банкротство

Доступно (`_can_liquidate`) либо в свой ход в фазе `awaiting_roll`, либо пока
висит `pending_debt` на этого игрока (`managing_assets`).

- **`request_build`** — требует полного владения группой, равномерной
  застройки внутри группы (`can_build`: нельзя строить на одном поле группы
  больше, чем на соседних), поле не заложено, максимум 5 уровней (4 дома +
  отель).
- **`request_sell_building`** — обратная операция, возвращает половину
  `house_cost`; тоже требует равномерности застройки внутри группы.
- **`request_mortgage`/`request_unmortgage`** — залог даёт `mortgage_value`
  (по умолчанию `price / 2`), выкуп стоит `mortgage_value * unmortgage_multiplier`
  (1.1 по умолчанию — 10% сверху).
- **`request_sell_property`** — только если в группе нет построек и поле не
  заложено; возвращает `sale_value` (по умолчанию 50% цены).
- **`request_bankruptcy`** — можно объявить только пока висит `pending_debt`
  именно на этого игрока. Все поля переходят кредитору (или в банк — с
  обнулением построек/залогов, если кредитор — банк, `peer 0`).

## Сделки (Trade)

Двусторонние, приватные (условия видят только участники, см.
`private_state()`). `propose_trade` доступен только в фазе `awaiting_roll`.
Валидация (`_validate_trade`) перепроверяется **дважды** — при предложении и
при принятии — на случай, если условия успели измениться (например, поле
уже застроено или продано другому). Деньги переводятся двумя `ledger.transfer`
с откатом первого перевода, если второй не прошёл (см. `respond_trade`).

## Победа

`_winner_peer()` — единственный оставшийся не-банкрот при `PHASE_ENDED`.
Партия заканчивается сразу, как только активных игроков остаётся 1 (кроме
`solo_test_mode`, где это тестовый одиночный режим и партия не завершается
автоматически).
