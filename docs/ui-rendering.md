# UI и рендер

## Экраны (`ui/`)

| Файл | Сцена | Назначение |
|---|---|---|
| `MainMenu.gd` | `ui/MainMenu.tscn` | Хост/подключение, выбор ruleset'а, вход в кастомизацию/настройки |
| `WaitingRoom.gd` | `ui/WaitingRoom.tscn` | Лобби: готовность, старт (только `table_owner`), без 3D-доски |
| `SettingsScreen.gd` | `ui/SettingsScreen.tscn` | Экран/качество эффектов/звук — общий для меню и лобби |
| `Hud.gd` | часть `Game.tscn` через `GameScene` | Игровой HUD: баланс, ход, контекстные действия по фазе |
| `TradeDialog.gd` | инстанцируется поверх HUD | Предложение/входящая сделка |
| `AssetDialog.gd` | инстанцируется поверх HUD | Управление активами (стройка/залог/продажа) |
| `TokenCustomizer.gd` | инстанцируется поверх меню/лобби | Живой 3D-превью фигурки в `SubViewport` |
| `MenuBackdrop.gd` | фон меню/лобби | Процедурный `_draw()`-фон: пылинки, силуэт доски |
| `LuxuryTheme.gd` + `UiPalette.gd` | — | Общая программная `Theme` и цветовая палитра для всех vanilla-экранов |

Каждый экран, кроме `Hud`, `TradeDialog`, `AssetDialog`, `TokenCustomizer`,
сам решает переход через `PackRegistry.resolve_scene(key, fallback)`, чтобы
DLC с `scene_overrides` могло подменить любой из них — см.
[modding.md](modding.md).

### Единый визуальный язык

`UiPalette` — палитра "тёмный банковский люкс" (фетр, красное дерево,
золото): `INK`, `FELT*`, `MAHOGANY*`, `GOLD*`, `IVORY`, `MUTED`, `DANGER`,
`SUCCESS`. `LuxuryTheme.create()` собирает из неё программную `Theme`
(без `.tres`-файлов) и применяется на каждом экране в `_ready()`. Кастомный
`Theme` — осознанный выбор: DLC-экраны могут просто переиспользовать
`LuxuryTheme.create()`, не таская за собой файлы ресурсов темы.

### HUD и контекстные действия

`Hud._refresh()` — единая точка, которая по `turn_phase` публичного снапшота
решает, какую панель действий показать (бросок / покупка / аукцион / тюрьма /
управление долгом / ожидание чужого хода). Кнопки добавляются динамически
через `_add_action`, старые чистятся `_clear_actions()` — нет статичной
разметки под каждую фазу, весь набор кнопок собирается заново при каждом
обновлении состояния.

## Рендер (`render/`)

| Файл | Назначение |
|---|---|
| `GameScene.gd` | Корень игровой сцены: окружение, свет, стол, камера, частицы, виньетка, HUD-слой |
| `BoardRenderer.gd` | Строит доску из клеток `BoardModel`, "карточки" полей, фигурки, кости, маркеры владения |
| `TokenView.gd` | Процедурная фигурка (или кастомная модель мода) с ореолом подсветки текущего игрока |

### Геометрия доски

Доска — квадрат `size × size`, играбельны только клетки периметра
(`BoardModel._perimeter_coords`, по часовой стрелке от `(0,0)`). Для
`size = 15` это `15*4 - 4 = 56` клеток. `TILE_SIZE` (1.6 мировых единиц) и
`CARD_MARGIN` (запас под "карточку" собственности с фото/ценой/лестницей
ренты) — общие константы, на которые опирается и `GameScene` при расчёте
размера стола и подгонке камеры (`_fit_camera`).

### "Карточка" поля собственности

`_build_property_card` — не просто подпись, а мини-карточка: иконка/лого
(снаружи, у края доски), затем строка названия, строка цены, затем **лестница
ренты** (`GameRules.rent_ladder`) — рента на каждом уровне владения группой
до полной застройки. Так игрок видит выгоду докупки поля группы ещё до
покупки, не открывая отдельное окно.

### Кастомные 3D-модели (моды)

`AssetLoader.instantiate_from_def(def, field)` — единая точка входа для
любой замены процедурной геометрии на модель мода: токены (`model`), декор
клетки (`decor_model`), кости и доска-подложка/стол ruleset'а
(`dice_model`/`board_model`/`table_model`). Всегда падает обратно на
процедурный вариант при ошибке — см. правило в
[architecture.md](architecture.md).

### Управление качеством

`GameConfig.effects_quality` (0 низкое / 1 сбалансированное / 2 кинематографичное)
и `GameConfig.reduced_motion` читаются напрямую в `GameScene`/`BoardRenderer`/
`TokenView` для условного включения bloom/SSAO/тумана/частиц/твинов — единого
"движка качества" нет, каждый эффект сам проверяет нужный флаг в месте
использования.
